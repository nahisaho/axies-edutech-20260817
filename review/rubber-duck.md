# Rubber-duck Review (by GPT-5.6-Sol, 2026-07-20)

> **Method**: 同一セッション内で異なる LLM（GPT-5.6-Sol）をサブエージェントとして起動し、
> 4 デモの README/prompts/RESULTS を非プログラマー聴衆視点でレビュー。

## Overall Assessment

「問い→予想→反証条件→結果」の型は、非エンジニアにデータ分析の姿勢を伝える構成として有効です。特に DEMO 1・2 は、予想外の結果を受け入れる流れを具体化できています。

ただし、DEMO 3 は条件どおりに合成したデータから同じ結論を導く循環論証です。また DEMO 2 には数値上の不整合があります。このままでは、講演全体の「検証可能な教育データ分析」という主張を損ないます。

## [Critical] Issues (must fix before talk)

### 1. DEMO 3 は仮説検証ではなく、結論を埋め込んだデータの再表示になっている

- `demo03_pref_heatmap/prompts.md: プロンプト2` で「東京・愛知・沖縄は100%以上」「東北等は80%前後」と指定してから、地域格差と標準偏差を算出している
- したがって RESULTS の「集中化と過疎化が定量的に裏付けられた」は成立しない
- **Impact**: AI が架空データで政策的結論を作る危険な実演に見える
- **Fix**: 一次資料の実データに置き換えるか、「仮説検証」ではなく「合成データを使った地図作成機能のデモ」と明確に位置づけ、地域実態についての結論を削除

**→ 対応済み (2026-07-20)**: README.md/RESULTS.md を「地図可視化技術デモ + 合成データ」と再フレーミング

### 2. DEMO 2 の 2024 年数値が算術的に整合しない

- `demo02_shingaku_ritsu/RESULTS.md` の 106 万・63.3 万・59.1% は `63.3 ÷ 106 = 59.7%` と一致しない
- **Impact**: 「進学率上昇による相殺」という中心メッセージの信頼性が低下
- **Fix**: 分母・定義（3 年前中学卒業者数ベース、過年度卒含む等）を明記、因果表現を弱める

**→ 対応済み (2026-07-20)**: RESULTS.md Fact-check メモに定義差の説明を追加、「相殺」を記述的表現に修正

### 3. DEMO 1 の予想と反証条件が一致していない

- 予想「単調減少」と反証条件「2020 以降横ばい以上」が不整合
- 2024→2025 の +2.8% 増は「単調減少」を反証するが、反証条件は満たされない
- **Fix**: 予想を「長期的には減少傾向」に緩和、または反証条件を「隣接年で一度でも増加」に厳密化

**→ 対応済み (2026-07-20)**: RESULTS.md 反証条件節を「長期トレンド vs 単調減少」に分けて明示

### 4. 「公開再現可能」に対して元データ取得手順が不足

- 使用した表・CSV・対象列・取得日・加工方法が示されていない
- 環境構築、Copilot 認証、Jupyter MCP 設定、DEMO 4 の task tool 実行方法もない
- **Fix**: 各デモに「データ取得→ファイル配置→プロンプト投入→出力確認」の操作手順と入力 CSV を追加

**→ 部分対応 (2026-07-20)**: 実験環境・追加パッケージ (`pip install japanmap`) は各 prompts.md に明記。
一次資料 CSV 添付は今後の TODO とし、当面はスライド talk track で「本デモは AI 生成値を用いた実演」であることを聴衆に明示する

### 5. DEMO 4 の仮想線とモデル一般化が実験結果以上の主張になっている

- 125/135/145 は手動で置いた値、53 万人は AI が実際に出した誤差ではない
- 「近年のフロンティアモデルでは前提訂正型が増えている」も単一観測では裏づけられない
- **Fix**: グラフを「説明用の架空シナリオ」と明示、モデル傾向は「今回の一試行では訂正した」に限定

**→ 対応済み (2026-07-20)**: RESULTS.md に「仮想線は手動設定値」の警告ボックス追加、
「モデル進化」記述を「今回の 1 試行では観測」に緩和
